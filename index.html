// evaluation rubric

Judging Criteria

Every submission reviewed by domain experts across five weighted dimensions with structured, actionable feedback.

25%
Innovation & Creativity
How original is the solution? Does it demonstrate a new approach or unique idea that pushes boundaries?
25%
Technical Complexity
Is the implementation robust, functional, scalable, and built with engineering best practices?
20%
Impact & Usefulness
Does the solution solve a real problem and add genuine, measurable value to its intended audience?
15%
User Experience
Is the product intuitive, visually polished, and does it effectively address user needs?
15%
Presentation & Demo
How clearly is the idea communicated? Is the demo compelling and the documentation thorough?
Submission Checklist
Working demo (video walkthrough or live link)
Public GitHub repo with clean, commented code
README: problem, solution, tech stack, setup
Architecture & design decisions doc
Slide deck / one-pager with impact summary
Team info + LinkedIn profiles of all members
// what to build

Requirements

Production-ready solutions demonstrating technical excellence and genuine real-world applicability.

🏗️ What to Build
  • Technical Excellence: Well-architected systems, efficient code, and scalable solutions.
  • Innovation & Creativity: Novel approaches, unique features, or new applications of technology.
  • Real-World Applicability: Demonstrated impact or potential for adoption in practical scenarios.
  • UX & Documentation: Clear UI/UX and supporting docs explaining the workflow and architecture.
  • Solutions can be web, mobile, cloud platforms, APIs, data pipelines, or full-stack applications.
📤 What to Submit
  • Project Title & Description: Clear summary of problem, solution, and approach (200–300 words).
  • Solution Demo: Screenshots, video walkthrough (3–5 min), or live demo link.
  • Technical Documentation: Architecture diagrams, code explanation, APIs, libraries, and frameworks.
  • Code Repository: GitHub/GitLab with clear instructions to run or test the solution.
  • Team Details: Names, roles, and LinkedIn profiles of all team members.
